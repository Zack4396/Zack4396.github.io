本文介绍了如何使用 stress-ng

获取源码

仓库地址：Github, 最新版本是 V0.17.07

下载

1

wget https://github.com/ColinIanKing/stress-ng/archive/refs/tags/V0.17.07.tar.gz

解压

1

tar -zxvf V0.17.07.tar.gz -C . && cd stress-ng-0.17.07

编译源码

编译 x86

1
2
3
4

make clean && make

# 查看生成文件
ls stress-ng

编译 aarch64

1
2

# 指定 对应平台的 GCC 路径
MY_CC=/opt/toolchain/aarch64/bin/aarch64-cros-linux-gnu-gcc
1
2
3
4
5
6
7
8
9

MY_HOST="$(echo $(basename $MY_CC) | sed 's/-gcc$//')"
MY_CXX="$(dirname $MY_CC)/$MY_HOST-g++"

# 编译报错: error: redefinition of ‘struct rseq’
# 需添加 'CFLAGS=-D__GLIBC_HAVE_KERNEL_RSEQ'
make clean && make CC=$MY_CC CXX=$MY_CXX CFLAGS=-D__GLIBC_HAVE_KERNEL_RSEQ

# 查看生成文件
ls stress-ng

命令行参数

/data # stress-ng --help
Usage: stress-ng [options]

--matrix [?]     启动 ? 个 worker 进行矩阵运算
-t [?]           设置测试超时时长 [?] 秒
-l [?]           设置CPU Loading 百分比 [?]%
--metrics-brief  启用指标并仅显示非零结果
--tz             从 thermal zone 收集温度数据
-v               启用详细输出

用途

测试 CPU

1
2
3
4
5
6
7

# --matrix         启动 0 个 worker 进行矩阵运算
# -t               设置测试超时时长为 1440m(24hrs) 或 720m(12hrs) 或 480m(8hrs)
# -l               设置 CPU 负载为 80%
# --metrics-brief  启用指标并仅显示非零结果
# --tz             从 thermal zone 收集温度数据
# -v               启用详细输出
stress-ng --matrix 0 -t 1440m -l 80 --metrics-brief --tz -v
